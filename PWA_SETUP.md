# PWA (Progressive Web App) Setup Guide

This document describes the PWA configuration for **ProbSol Materialised**.

## Overview

ProbSol is fully configured as a PWA and can be installed on any platform that supports web standards:
- **iOS**: via Add to Home Screen (Safari)
- **Android**: via Add to Home Screen (Chrome, Firefox, Samsung Internet)
- **Windows**: via Microsoft Edge or Chrome
- **macOS**: via Safari or Chrome

## Configuration Files

### Web Manifest
- **File**: Generated by Vite PWA plugin as `dist/manifest.webmanifest`
- **Configured in**: `vite.config.ts`
- **Key settings**:
  - `display: 'standalone'` — App runs in fullscreen without browser UI
  - `display_override: ['standalone', 'minimal-ui', 'browser']` — Fallback modes for older browsers
  - `start_url: '/'` — Opens app at root
  - `scope: '/'` — App scope for navigation handling
  - `theme_color: '#4f46e5'` (light) / `'#8b5cf6'` (dark) — Browser chrome color
  - `background_color: '#f7f8fb'` — Splash screen background

### App Icons
- **Location**: `public/` directory
- **Icon set**:
  - `favicon.svg` (48x48) — Primary favicon
  - `pwa-192x192.png` — Main app icon
  - `pwa-512x512.png` — Large app icon
  - `maskable-icon-512x512.png` — Adaptive icon for Android (with safe zone)
  - `apple-touch-icon.png` (180x180) — iOS home screen icon

### Apple Splash Screens
- **Location**: `public/splash/` directory
- **Variants**: 
  - Light mode: 25 different device/orientation combinations
  - Dark mode: 25 different device/orientation combinations
- **Coverage**: iPhone, iPad, iPad Air, iPad Pro, iPad mini (all modern sizes)
- **Configured in**: `index.html` with `<link rel="apple-touch-startup-image">` tags

### Meta Tags
- **File**: `index.html`
- **Key meta tags**:
  ```html
  <meta name="color-scheme" content="light dark">
  <meta name="theme-color" content="#4f46e5" media="(prefers-color-scheme: light)">
  <meta name="theme-color" content="#8b5cf6" media="(prefers-color-scheme: dark)">
  <meta name="mobile-web-app-capable" content="yes">
  <meta name="apple-mobile-web-app-capable" content="yes">
  <meta name="apple-mobile-web-app-status-bar-style" content="black-translucent">
  ```

### Service Worker
- **Generated by**: `vite-plugin-pwa` 
- **Location**: `dist/sw.js`
- **Features**:
  - Auto-updates via `registerType: 'autoUpdate'`
  - Precaches 63+ assets (JS, CSS, HTML, icons, splash screens)
  - Network fallback to `/index.html` for SPA routing
  - Workbox-powered caching strategies

### Windows Configuration
- **File**: `public/browserconfig.xml`
- **Purpose**: Configures Windows Start menu tiles
- **Tile color**: #4f46e5 (brand accent)

## Installation Instructions

### iOS (Safari)
1. Open https://probsol.vercel.app (or your deployed URL) in Safari
2. Tap the **Share** button (square with arrow)
3. Scroll down and tap **Add to Home Screen**
4. Enter app name: "ProbSol" (optional)
5. Tap **Add** in top-right

### Android (Chrome / Firefox)
1. Open https://probsol.vercel.app in your browser
2. Tap the **menu** (three dots) or **address bar**
3. Tap **Add to Home screen** or **Install app**
4. Confirm the installation

### Windows (Edge / Chrome)
1. Open https://probsol.vercel.app
2. Click the **install** icon (down arrow) in the address bar
3. Click **Install** in the dialog
4. App opens in a standalone window

### macOS (Safari / Chrome)
1. Open https://probsol.vercel.app in Safari
2. Go to **File → Add to Dock** (Safari 15+)
3. Or in Chrome, click the **install** icon in the address bar

## Testing PWA Features

### Test Installation
```bash
npm run build
npm run preview
```
Then open http://localhost:4173 and test install prompts.

### Test Service Worker
1. Open DevTools (F12)
2. Go to **Application → Service Workers**
3. Verify `sw.js` is registered and active
4. Go offline and navigate — app should still work

### Test Splash Screens (iOS)
1. Add app to home screen on iOS
2. Launch it
3. Verify the splash screen loads before the app

### Test Icons
1. Check home screen icon appears correctly
2. Check notification icons load
3. On Android, verify adaptive icon displays properly

### Test Cache
1. Build and run locally
2. Open DevTools → **Application → Cache Storage**
3. Verify cached assets are stored

## Updating the PWA

After code changes:

```bash
npm run build
```

The `vite-plugin-pwa` will:
1. Generate a new manifest
2. Create a new service worker
3. Update the precache list automatically
4. Service worker auto-updates clients

Users will see updates within 24 hours (or on next app launch if configured for immediate update).

## Troubleshooting

### App Not Installable
- Check manifest is accessible: https://yoursite.com/manifest.webmanifest
- Verify icons exist and are accessible
- Check HTTPS is enabled (PWA requires HTTPS on production)
- Verify display mode is `standalone`

### Splash Screen Not Showing on iOS
- Verify Apple splash image sizes and media queries match device specs
- Check file names follow naming convention: `apple-splash-WIDTHxHEIGHT.png`
- Verify media queries in `<link>` tags are correct

### Service Worker Not Updating
- Open DevTools → **Application → Service Workers**
- Click **Unregister** to force clear
- Hard refresh (Cmd+Shift+R / Ctrl+Shift+R)
- Rebuild and deploy
